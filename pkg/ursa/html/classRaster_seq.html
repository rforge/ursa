seqR Documentation

Sequence Generation for raster image and coordinate grid

Description

Set of functions to generate regular sequences of bands, x-/y-cordinates and columns/rows.

Usage

## S3 method for class 'rmapRaster'
seq(...)

## S3 method for class 'rmapGrid'
seq(...)

rmap.seqx(obj)
rmap.seqy(obj)
rmap.seqc(obj)
rmap.seqr(obj)

Arguments

...

Set of argumetns, which are recognized via their names (using regular expressions), position and classes.

.*

First argument (position 1). Object of classes rmapRaster, rmapGrid.

.*

Second argument (position >1). One-character name. Valid values are in the list c("z","x","y","c","r","lines","samples"). "c" ("samples") and "r" ("lines") specify to generate cell sequence in the horizontal and vertical directions from bottom-left corner, whereas "z" specifies to generate sequence of bands. x and "y" return cell midpoints in spatial dimension.

obj

Object of classes rmapRaster, rmapGrid. Missing obj is allowed; in this case the session grid is considered.

Details

All ordinal sequences (axis is \dQuote{c}, \dQuote{r}, \dQuote{z}) start from 1L.

axis=\dQuote{z} is ignored in the function seq for rmapGrid object. The returned value is 1L.

seq(obj) for rmapRaster objects is suitable for using in cycles across bands.

Value

Functions rmap.seqx and seq(obj, "x") return x-coordinates of cell midpoints.
Functions rmap.seqy and seq(obj, "y") return y-coordinates of cell midpoints.
Functions rmap.seqc, seq(obj, "samples") and seq(obj, "c") return sequence of cells in horizontal direction.
Functions rmap.seqr, seq(obj, "lines") and seq(obj, "r") return sequence of cells in vertical direction.
Functions seq(obj) and seq(obj, "z") for rmapRaster object returns sequence of bands.
Function seq(obj) and seq(obj, "z") for rmapGrid object returns 1L.

Examples

session.grid(NULL)
session.grid(regrid(mul=1/16))
print(session.grid())
a <- rmap.dummy(nband=5)
print(a)
print(seq(a))
print(seq(a,"c"))
print(seq(a,"x"))
