rmapStackR Documentation

List of raster images.

Description

Functions to create list (layers) of multiband raster images (stack in the notation of raster package) and to coerce list of images to single multiband image (brick in the notation of raster package).

Usage


rmap.stack(...)

rmap.brick(obj)

rmap.apply(obj, FUN, ...)

## S3 method for class 'rmapRaster'
as.list(x, ...)

## S3 method for class 'rmapStack'
unlist(x, recursive, use.names)

Arguments

obj, x

Object of class rmapRaster or list of rmapRaster objects. In function rmap.apply argument "X", which is passed to function lapply.

FUN

Argument "FUN", which is passed to function lapply.

recursive

Not used. For consistency with generic function unlist.

use.names

Not used. For consistency with generic function unlist.

...

Denending of functions:

rmap.stack, as.list
List of rmapRaster objects
rmap.apply
Arguments "...", which are passed to function lapply.

Details

as.list (of rmapRaster object(s)), rmap.stack create list of rmapRaster objects, where items of list are sinle-band images. If x is rmapRaster object, then list(x) create a list of length one, which item is multiband image.

unlist (for list of rmapRaster objects), rmap.brick create single multiband rmapRaster object. There is an alternative way for unlisting the list of rmapRaster: as.rmap.

Raster stack is a way to group bands, for example, by units (degree Celsium, meters).
Raster brick is a way to combine versalite images to the single multiband image, for example, for saving in file.

Value

rmap.stack, as.list return object of class rmapStack. It is a list, with class "rmapStack" attribute.

unlist (for list of rmapRaster objects), rmap.brick return object of class rmapRaster.

rmap.apply returns object of class rmapStack, if result is list of rmapRaster objects, otherwise returns general list.

Warning

There is no any verifications, that grids of rmapRaster objects are the same.

Note

Generic unlist(x) deals only with class of x, but doesn't take into account class of objects in list (e. g., x[[1]]). So, there is no effective way to use only list/unlist for rmapRaster objects to do a conversion between raster brick and stack. Examples

session.grid(NULL)
a <- rmap.dummy(3)
print(a)
b1 <- rmap.stack(a[1:2],colorize(a[3],ramp=FALSE))
print(b1)
b2 <- as.list(a)
print(b2)
b3 <- list(a[1],a[2:3])
print(b3)
b31 <- lapply(b3,colorize,ramp=FALSE)
print(b31)
b32 <- rmap.apply(b3,colorize,ramp=FALSE,rev=TRUE)
print(b32)
s311 <- rmap.apply(b31,rmap.colortable)
print(s311)
s21 <- lapply(b2,global.mean)
print(s21)
s22 <- sapply(b2,global.mean)
print(s22)
s31 <- lapply(b3,global.mean)
print(s31)
s32 <- sapply(b3,global.mean)
print(s32)
c1 <- unlist(b1)
print(c1)
c2 <- unlist(b2)
print(c2)
c3 <- unlist(b3)
print(if (is.rmap(c3)) c3 else "broken object")
d3 <- as.rmap(b3)
print(if (is.rmap(d3)) d3 else "broken object")
